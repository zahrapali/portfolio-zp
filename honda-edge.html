Honda Edge App

Role:

Product Designer + Front-end Developer.

OBJECTIVE:

To create a tool to improve the in-person and online user experience while shopping, buying and delivering a car at the Honda Dealerships.

PLATFORM:

React + WordPress Front-End.

User Types Included:

  1. Sale Consultants
  2. Service Consultants
  3. Managers
  4. HQ team

User Research:

Purchasing a car is a very personal experience. Learning how honda employees could support this process and provide a tool to enhance the experience was our objective while performing our user research. Many stakeholders mean we had to build out several user journeys.

Challenges:

Student ID. The Student ID was a unique sequence number provided by the Acura HR Management team. This input while creating an account was a requirement for the stakeholders. However, most employees don't know this ID by heart. We needed a way to increase retention within new accounts, while also ensuring individuals would complete the onboarding. We decided we needed to create an easy informational button. This showed an increase in successful onboards by 40%.
